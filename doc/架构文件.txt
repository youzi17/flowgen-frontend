# AI工作流可视化编排平台 - 完整架构文档 (Element Plus版本)

## 📋 项目概述

**项目名称**：AI工作流可视化编排平台（纯前端版本）  
**技术栈**：Vue 3 + TypeScript + Pinia + **Element Plus** + Vue Flow + LocalForage  
**项目目标**：构建完整的纯前端工作流编排器，支持本地存储和完整的工作流管理

---

## 🏗️ 整体架构设计

### 项目结构
```
src/
├── components/          # 通用组件
│   ├── common/         # 基础UI组件
│   ├── workflow/       # 工作流相关组件
│   └── nodes/          # 节点类型组件
├── views/              # 页面组件
├── stores/             # Pinia状态管理
├── types/              # TypeScript类型定义
├── utils/              # 工具函数
├── hooks/              # Vue 3组合式函数
├── core/               # 核心引擎
│   ├── workflow-executor.ts
│   ├── parameter-resolver.ts
│   ├── node-registry.ts
│   └── storage-manager.ts
├── constants/          # 常量定义
└── assets/             # 静态资源
```

## 🎯 修改总结

### 主要变更：
1. **UI组件库**：Ant Design Vue → Element Plus
2. **图标系统**：使用 `@element-plus/icons-vue`
3. **布局组件**：`a-layout` → `el-container` 系列
4. **表单组件**：`a-form` → `el-form` 系列
5. **弹窗组件**：`a-modal`/`a-drawer` → `el-dialog`/`el-drawer`
6. **消息提示**：`message` → `ElMessage`

### 保持不变：
- 所有业务逻辑
- 状态管理 (Pinia)
- 核心引擎 (工作流执行、存储管理、节点注册)
- 类型定义
- 项目结构
- 数据流设计

这样的修改确保了项目的核心功能完全不受影响，只是UI层从Ant Design迁移到Element Plus，保持了高置信度的架构设计。
