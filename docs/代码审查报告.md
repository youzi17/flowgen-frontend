## 🚨 严重问题 (必须修复)

### 1. 大量使用 `any` 类型 (违反项目规范)

**位置**: 多个文件

**问题描述**:
项目中大量使用了 `any` 类型，违反了 CLAUDE.md 中明确规定的"TypeScript里面绝对不能使用any"。

**受影响文件**:
- `src/core/nodes/DataProcessNode.ts`: 5处
- `src/core/nodes/DataProcessNodeProcessor.ts`: 17处
- `src/core/nodes/data-process/index.ts`: 1处
- `src/core/nodes/index.ts`: 1处
- `src/core/nodes/registerDataProcessNode.ts`: 4处
- `src/ui/canvas/WorkflowCanvas.vue`: 3处
- `src/ui/components/VariableSelector.vue`: 2处
- `src/ui/nodes/ai-chat-node.vue`: 1处

**影响**:
- 失去TypeScript类型安全保护
- 可能导致运行时错误
- 违反团队开发规范

**建议修改**:
```typescript
// ❌ 当前代码
const data: any = {}
function process(input: any): any {}

// ✅ 建议改进
interface ProcessData {
  id: string
  value: number
}

const data: ProcessData = { id: '', value: 0 }
function process(input: ProcessData): ProcessData {}
```

**观察**: `WorkflowCanvas.vue` 有252行代码，接近建议的300行上限

**建议**: 考虑将拖拽逻辑提取为组合式函数:

```typescript
// composables/useDragAndDrop.ts
export function useDragAndDrop() {
  const isDragOver = ref(false)

  const onDrop = (event: DragEvent) => {
    // 拖拽逻辑
  }

  const onDragOver = (event: DragEvent) => {
    // 悬停逻辑
  }

  return {
    isDragOver,
    onDrop,
    onDragOver,
    onDragEnter,
    onDragLeave
  }
}
```

**建议**: 添加加载状态和错误处理:
