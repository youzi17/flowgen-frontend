# AI工作流可视化编排平台 - 模块化架构设计 (Element Plus版本)

## 🏗️ 模块划分架构

### 模块依赖关系图
```
App (入口)
├── Core (核心模块) - 保持不变
│   ├── WorkflowEngine (工作流引擎)
│   ├── StorageManager (存储管理)
│   └── NodeRegistry (节点注册)
├── Store (状态管理) - 保持不变
│   ├── WorkflowStore
│   ├── UIStore
│   └── HistoryStore
├── UI (界面模块) - Element Plus
│   ├── Layout (布局)
│   ├── Canvas (画布)
│   ├── NodePalette (节点面板)
│   ├── ConfigPanel (配置面板)
│   └── Toolbar (工具栏)
└── Nodes (节点模块) - 保持不变
    ├── BaseNode (基础节点)
    ├── AINodes (AI节点)
    ├── LogicNodes (逻辑节点)
    └── DataNodes (数据节点)
```

export class WorkflowExecutor {
  private context: ExecutionContext = {};
  
  async execute(nodes: WorkflowNode[], edges: Edge[]): Promise<ExecutionContext> {
    // 拓扑排序和执行逻辑 - 完全保持不变
    const sortedNodes = this.topologicalSort(nodes, edges);
    
    for (const node of sortedNodes) {
      await this.executeNode(node, edges);
    }
    
    return this.context;
  }

}

#### 3.2 Canvas (画布组件)
```vue
<!-- ui/canvas/WorkflowCanvas.vue -->
<template>
  <VueFlow
    v-model:nodes="workflow.nodes"
    v-model:edges="workflow.edges"
    @node-click="onNodeClick"
    @connect="onConnect"
    @drop="onDrop"
    @drag-over="onDragOver"
  >
    <Background />
    <Controls />
    
    <!-- 动态节点渲染 - 保持不变 -->
    <template v-for="nodeType in nodeTypes" #[`node-${nodeType}`]="props">
      <component 
        :is="getNodeComponent(nodeType)" 
        v-bind="props" 
      />
    </template>
  </VueFlow>
</template>

<script setup lang="ts">
import { useWorkflowStore } from '@/stores/workflow-store';
import { useUIStore } from '@/stores/ui-store';
import { useHistoryStore } from '@/stores/history-store';

// 业务逻辑完全保持不变
const workflow = useWorkflowStore();
const ui = useUIStore();
const history = useHistoryStore();

const nodeTypes = ['start', 'ai-chat', 'condition', 'end'];

const getNodeComponent = (type: NodeType) => {
  return defineAsyncComponent(() => 
    import(`@/ui/nodes/${type}-node.vue`)
  );
};

const onNodeClick = (event: NodeMouseEvent) => {
  ui.selectNode(event.node.id);
};

const onConnect = (connection: Connection) => {
  workflow.addEdge(createEdge(connection));
  history.pushState();
};

const onDrop = (event: DragEvent) => {
  const type = event.dataTransfer?.getData('application/node-type');
  const position = { x: event.clientX, y: event.clientY };
  
  if (type) {
    workflow.addNode(type as NodeType, position);
    history.pushState();
  }
};

const onDragOver = (event: DragEvent) => {
  event.preventDefault();
  event.dataTransfer!.dropEffect = 'move';
};
</script>
```

  ChatDotRound,
  Operation,
  CircleClose,
  DataLine

const iconMap = {
  'start': VideoPlay,
  'ai-chat': ChatDotRound,
  'condition': Operation,
  'end': CircleClose,
  'data-process': DataLine
};

const activeCategories = ref(['ai', 'logic']);

// 业务逻辑保持不变
const categories = computed(() => {
  const allNodes = NodeRegistry.getAllDefinitions();
  const grouped: Record<string, NodeDefinition[]> = {};
  
  allNodes.forEach(node => {
    if (!grouped[node.category]) {
      grouped[node.category] = [];
    }
    // 添加图标映射
    grouped[node.category].push({
      ...node,
      icon: iconMap[node.type] || Operation
    });
  });
  
  return Object.entries(grouped).map(([name, nodes]) => ({
    name: getCategoryName(name),
    nodes
  }));
});

const onDragStart = (nodeDef: NodeDefinition, event: DragEvent) => {
  event.dataTransfer?.setData('application/node-type', nodeDef.type);
};

  const names: Record<string, string> = {
    'ai': 'AI节点',
    'logic': '逻辑节点',
    'data': '数据节点',
    'basic': '基础节点'
  };
  return names[category] || category;
};

## 🔄 数据流设计 - **完全保持不变**

### 数据流动示意图
```
用户操作 → UI组件 → Store Action → Core模块 → Store Mutation → UI更新
     ↓
   状态变化 → 历史记录 → 本地存储
```

### 典型数据流示例 - 保持不变

#### 1. 添加节点数据流 - 保持不变
```
1. 用户从NodePalette拖拽节点
2. NodePalette发出dragstart事件  
3. WorkflowCanvas监听drop事件
4. Canvas调用workflowStore.addNode()
5. Store创建新节点并更新状态
6. Store调用historyStore.pushState()
7. UI自动重新渲染显示新节点
```

#### 2. 执行工作流数据流 - 保持不变
```
1. 用户点击执行按钮
2. Toolbar调用workflowStore.executeWorkflow()
3. Store创建WorkflowExecutor实例
4. Executor进行拓扑排序并依次执行节点
5. 每个节点通过NodeRegistry获取执行器
6. 执行结果更新到executionContext
7. UI通过computed属性实时显示执行状态
```
